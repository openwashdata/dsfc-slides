What is ggplot2?

ggplot2 is an R package for data visualization based on the Grammar of Graphics.

It lets you build plots by combining components in a structured way, like building sentences.

Grammar of Graphics

Example:

“Make a bar chart of sales by region and products.”

In a structured way, translates to:

“Using the sales data, put region on the x-axis, sales on the y-axis, and group bars by product with a unique color for each region.”

Each part of the sentence maps to a component of the plot.

Essential building blocks

The three essentials

  1. Data: a data.frame or tibble, passed to ggplot()
  2. Aesthetics: how variables map to visual properties (position, color, size), defined inside aes()
  3. Layers: where data takes shape (points, lines, bars), added with geom_*() or stat_*() functions

Good to know

With just these three elements you can already generate a plot!

Additional components

Scales

Scales fine-tune how data is visually represented.

With scale_*() functions you can:

  • Change color palettes and shapes
  • Modify axis labels and limits
  • Control legend behavior

Coordinates

Coordinates define the spatial context of the plot.

With coord_*() functions you can:

  • Apply spatial transformations (e.g., coord_flip())
  • Set aspect ratios
  • Zoom into a region of the plot

Facets

Facets split a plot into multiple panels based on subsets of the data.

  • facet_wrap(): panels from one variable
  • facet_grid(): panels from two variable combinations

Useful to uncover patterns across groups!

Themes

Themes control the overall visual style of the plot.

  • theme_*() functions apply a complete style (e.g., theme_minimal())
  • theme() lets you fine-tune individual elements (text, backgrounds, gridlines)

The full “sentence”

ggplot(data = mpg, aes(x = displ, y = hwy, color = class)) +
  geom_point() +
  scale_x_continuous(name = "Engine displacement") +
  scale_y_continuous(name = "Highway MPG") +
  scale_color_brewer(name = "Class", palette = "Set2") +
  facet_wrap(~drv) +
  theme_minimal()

Choosing the right plot

What plot should I use?

It depends on the type of variable you want to visualize.

Types of variables

Non-numerical

  • Categorical: no inherent order (e.g., color, country)
  • Ordinal: with a meaningful order (e.g., education level, rating)

Numerical

  • Discrete: countable values (e.g., number of children)
  • Continuous: any value in a range (e.g., temperature, weight)

Can you tell the type?

Does it make sense to do math with this variable?

Zip code: 10115, 28001, 75008…

Looks like a number, but…

What is the average zip code?

It makes no sense. Zip codes are categorical.

Year: 2019, 2020, 2021…

It depends on the context!

  • Comparing groups (sales in 2020 vs 2021): categorical
  • Tracking change over time: continuous

The same variable can be treated differently depending on your question.

Likert scale: 1, 2, 3, 4, 5

A satisfaction survey asks: “Rate from 1 (very bad) to 5 (excellent).”

Is the difference between 1 and 2 the same as between 4 and 5?

Not necessarily. These are ordinal, not truly numerical.

Age group: “18-25”, “26-35”, “36-45”…

The values contain numbers, but…

You can’t average “18-25” and “26-35”.

These are ordinal: categories with a meaningful order.

“How many hours did you sleep?” 4, 6, 7, 8…

Looks continuous, but people usually answer in whole numbers.

Is “7 hours” really 7.0, or just a rounded estimate?

In a survey, this is arguably discrete: the responses are countable, even if the underlying variable is continuous.
