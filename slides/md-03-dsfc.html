Base R vs packages

What is base R?

Base R is the set of functions and tools that come installed with R itself.

It includes:

  • Core data types: vectors, lists, data frames
  • Basic operations: arithmetic, logic, subsetting
  • Built-in functions: mean(), sum(), length(), summary()
  • Built-in datasets: iris, mtcars, airquality

What is a package?

A package is a collection of functions, data, and documentation that extends R’s capabilities.

  • Developed by the R community
  • Installed from CRAN, GitHub, or other sources
  • Must be loaded before use

Installing and loading packages

Install a package once with install.packages():

install.packages("dplyr")

Load it in every session with library():

library(dplyr)

The tidyverse

The tidyverse is a collection of packages designed for data science that share a common philosophy.

Core packages include: ggplot2, dplyr, tidyr, readr, tibble, purrr, stringr, forcats, lubridate.

library(tidyverse)

R building blocks

Objects

An object is anything you store in R’s memory with a name.

x <- 10
name <- "Adriana"
numbers <- c(1, 2, 3, 4, 5)

Once created, you can use the name to access the value.

numbers
[1] 1 2 3 4 5

Functions

A function is a reusable block of code that performs a task.

mean(numbers)
[1] 3

Functions are called by their name followed by parentheses: function_name().

Arguments

Arguments are the inputs you pass to a function.

round(3.14159, digits = 2)
[1] 3.14

Arguments can be named (digits = 2) or positional. Named arguments make your code clearer.

The assignment operator

Use <- to assign a value to an object.

age <- 30
age
[1] 30

You can also use =, but <- is the convention in R.

Keyboard shortcut

Alt + - (Windows/Linux) or Option + - (Mac) inserts <-.

The pipe operator

The pipe takes the result on the left and passes it as the first argument to the function on the right.

Without a pipe:

round(mean(c(1, 2, 3, 4, 5)), digits = 2)
[1] 3

With a pipe, the same operation reads left to right.

Two pipe flavors

magrittr pipe %>%

Comes from the magrittr package, loaded with the tidyverse.

c(1, 2, 3, 4, 5) %>%
  mean() %>%
  round(digits = 2)
[1] 3

base pipe |>

Built into R since version 4.1 (2021). No package needed.

c(1, 2, 3, 4, 5) |>
  mean() |>
  round(digits = 2)
[1] 3

Which pipe to use?

  • For new code, prefer the base pipe |>: no dependencies, part of R itself
  • You will see %>% in older code and tutorials: they behave almost the same for our purposes

Keyboard shortcut

Ctrl + Shift + M (Windows/Linux) or Cmd + Shift + M (Mac) inserts a pipe. You can set which one in RStudio preferences.

Introducing dplyr

What is dplyr?

dplyr is the tidyverse package for data processing.

It gives you a consistent set of verbs to manipulate data frames: filter rows, select columns, create new variables, compute summaries, and more.

The four dplyr rules

  1. The first argument is always a data frame
  2. The next arguments describe what to do with that data frame
  3. It returns a data frame
  4. It does not modify the original data frame

These rules are what make dplyr play beautifully with the pipe.

The big picture

Verbs at a glance

Verb What it does
filter() Keep rows that match a condition
select() Keep or drop columns
arrange() Sort rows
relocate() Reorder columns
rename() Rename columns
mutate() Create or modify columns
transmute() Create columns, drop the rest
summarize() Collapse to summary statistics
group_by() Compute per group
count() Shortcut for grouped counts

Putting it all together

starwars |>
  filter(!is.na(height), !is.na(mass)) |>
  mutate(bmi = mass / (height / 100) ^ 2) |>
  group_by(species) |>
  summarize(avg_bmi = mean(bmi), n = n()) |>
  arrange(desc(avg_bmi))
